# MVP Experiment Plan — FoodFast

## 1. Experiment Objective
Our objective is to test whether campus students can seamlessly select menu items, successfully complete a simulated checkout transaction, and monitor live order tracking states without assistance.

## 2. Selected Experiment Type
* **Selected Method:** Simple Web Prototype / Interactive Flow.
* **Justification:** Since core usability and transactional state management represent our highest systemic risks, a clickable interface using simulated database records is the most realistic framework to collect objective user interaction data.

## 3. Targeted Test Users
* **Cohort:** 5 active university students who regularly purchase lunch at campus canteens.
* **Selection Criteria:** Must match the student customer persona defined in Lab 04.

## 4. Traceability Mapping
Our MVP experiment maps directly to the following development parameters:
* **System Requirements:** `FR-01` (Menu View), `FR-02` (Cart Operations), and `FR-03` (Live Status Updates).
* **User Stories:** `US-03` (Browse stalls), `US-05` (Customized checkout), and `US-09` (Receive status updates).
* **Screens Tested:** Customer Menu View, Shopping Cart Checkout, Live Tracking Screen.
