# Feature-Value Mapping — FoodFast

This document maps our Minimum Viable Product (MVP) features back to user requirements, business value, and specific user stories.

| Feature ID | Feature Name | Connected Req ID | User Story ID | User Value | Business Value | Target Screen/Module |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **FT-01** | Vendor Menu Browsing | REQ-01 | US-03 | Students can see all available meals and real-time prices without walking to canteens. | Increases customer engagement and drives digital order volume. | `Home Screen / Menu View` |
| **FT-02** | Cart & Checkout | REQ-02 | US-05 | Quick, customized item gathering with instant pricing summary. | Drives conversion and converts browsing intent into concrete actions. | `Cart Screen` |
| **FT-03** | Live Order Tracking | REQ-03 | US-09 | Eradicates physical waiting lines; students know exactly when to pick up food. | Enhances operational transparency and drops canteen crowd congestion. | `Order Status View` |
| **FT-04** | Vendor Dashboard | REQ-04 | US-12 | Vendors view incoming tickets sequentially and update statuses seamlessly. | Simplifies merchant kitchen workflows and optimizes meal prep times. | `Merchant Admin Portal` |
