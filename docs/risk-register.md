# Risk Register Summary — FoodFast

This summary tracks key operational, technical, and privacy risks associated with the FoodFast ecosystem.

## Top Identified Risks
1. **RSK-04 (Database Overload during Lunch Hours):** High risk of transaction failure between 11:00 AM – 1:00 PM. Mitigated via auto-scaling and cached menu states.
2. **RSK-01 (Student Phone Number Exposure):** Medium privacy risk. Mitigated by masking phone numbers on public vendor kitchen displays.
3. **RSK-02 (Unauthorized Canteen Vendor Access):** Security risk. Mitigated using credential checks and protected backend routes.
